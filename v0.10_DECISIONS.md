# ZCC v0.10 Architectural Decisions

This document tracks decisions made leading up to the v0.10 stabilization around the integration of novel floating-point formats (`TOFP32`, `EMFP16_LIVE`, `SPSQ32_DIFF`) and their telemetry-driven coercion infrastructure.

## 1. Cross-format Coercion: Option C Selected
During v0.10, we implemented deep telemetry hooking to measure live empirical demand for cross-format novel casting (e.g. `TOFP32` to `EMFP16_LIVE`). While testing yielded authentic signals from the stress harnesses, we struck an architectural dilemma:

The compiler can analytically rank operations, but *the coercion isn't a pure compiler transform*. Formats like `EMFP16_LIVE` require runtime context that doesn't exist at compile IR-time (like the state of the on-chip SRAM live H-field). Setting `cost < 255` in the backend caused ZCC to emit `COERCE_OK` telemetry while subsequently emitting fail-closed fallback codegen because `libzkaedi_fpx.a` doesn't exist yet to actually implement the conversion.

This resulted in **Option C** being finalized for v0.10:
* **All novel↔novel cross-casts are treated as parse errors.** You cannot write `(_EMFP16_LIVE) tofp_val`.
* The `coerce_cost_table` has been fully reverted so these edges reflect `255 (UNIMPL)`. It serves internally as a documentation artifact for what operations the silicon/runtime *will* provide.
* This safely bounds the compiler scope and guarantees telemetry integrity. We will reopen these operations in **v0.11** when `__rt_coerce_*` runtimes are formally designed.

## 2. Telemetry Sensitivity & The Node-Line Upgrade
The initial `ZCC_EMIT_CAST` telemetry layer assigned a global, hardcoded Context Hash for each edge type (`src << 16 | dst`). This clustered every cast—even spanning hundreds of independent lines—into a single unique site, causing the Forensics Spread Ratio metric (`uniq_ctx / events`) to falsely flag real operations as phantom pointer arithmetic array contamination (Hypothesis 2).

* **Permanent Plumbing Upgrade:** Context Hash reporting has been upgraded from static binding to using the AST's exact `node->line`. Context Hashes now deterministically capture precise source origins.
* Because the stress harness population mapped only 4 cast configurations against ~27 iterations, the density was severely lacking statistical power for confident codegen tuning. Reverting to Option C buys time to build larger telemetry aggregates.

## 3. Narrowing the Telemetry Hooks
Now that novel↔novel logic is relegated to AST parser rejection, the telemetry hook `ir_tel_fpx_coerce_cast_attempt()` was narrowed specifically. 
It *only* fires on `novel ↔ standard` implementations, preventing uncompilable operations from polluting the heatmap datasets and providing pure signal extraction for the next update cycle.

## 4. `libzkaedi_fpx.a` Design Brief for v0.11
The upcoming task for v0.11 is transitioning ZKAEDI PRIME's numerical primitives into full compiler-to-runtime dispatch.
`libzkaedi_fpx.a` will inject the physics bridges required to finalize these formats:
```c
EMFP16_LIVE __rt_coerce_tofp32_to_emfp16_live(
    TOFP32 src, uint32_t pixel_ctx, float *H_live);
```
Once `__rt_coerce_*` structures are available:
1. `ir_cast_dispatch_existing()` will generate `call __rt_coerce_*` instead of fail diagnostics.
2. `coerce_cost_table` edges will be unsealed to their true silicon gate estimates.
3. The `part3.c` parser safety lock will be dissolved.
