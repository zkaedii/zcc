Phase 2 — Arena Probe & Limit Register
Trap #1 — Arena discovery. The bounds check needs an upper limit to compare against. Three strategies with different trade-offs — pick one before writing any injection code.
Strategy A — %r15 as Safety Register (Recommended)
Reserve %r15 as a dedicated "arena ceiling" register. Load it once at program entry via brk(0) (returns current heap break = top of heap). Every bounds check is then a single cmpq %rax, %r15. Fastest check possible — 1 cycle latency.
Trap #2 — Stack pointer aliasing. Stack addresses are ABOVE %r15 (heap grows up, stack grows down from top of address space). A naive cmpq %rax, %r15; ja .trap would flag every stack-resident pointer as OOB. Solution: only inject checks for pointers proven to be heap-derived at AST level, OR use a two-sided range check (lo bound = program text end, hi bound = heap break).
  • In codegen_func() prologue, when sanitize=1: emit pushq %r15 (callee-save), then load heap break into %r15
    part4.c codegen_func() MEDIUM
  • In codegen_func() epilogue: emit popq %r15 before every retq
    %r15 is callee-saved in SysV ABI. Pushed/popped every function that's compiled with T8 active.
    part4.c codegen_func() MEDIUM
; T8 function prologue injection (when cc->sanitize)
pushq %r15                     ; callee-save %r15
xorq  %rdi, %rdi              ; brk(0) = query current break
movq  $12, %rax               ; syscall: brk
syscall
movq  %rax, %r15              ; %r15 = heap ceiling

; T8 function epilogue injection
popq  %r15                     ; restore %r15 before ret
retq
Strategy B — .zcc_arena_hi Symbol (Simpler, No %r15 Reserve)
Emit a global .zcc_arena_hi symbol in the output file's preamble. The sanitized main() initializes it via brk(0) at startup. Each check does a memory load (cmpq .zcc_arena_hi(%rip), %rax) — slightly slower than Strategy A (memory load vs register), but doesn't require reserving %r15.
; Preamble emission (once, at start of output file)
    .data
.zcc_arena_hi:   .quad 0       ; initialized at runtime
    .text

; main() injection (once, when sanitize=1)
xorq  %rdi, %rdi
movq  $12, %rax               ; brk(0)
syscall
movq  %rax, .zcc_arena_hi(%rip)

; Per-check (every pointer write)
cmpq  .zcc_arena_hi(%rip), %rax
ja    .zcc_sanitize_trap
Phase 3 — Crash Handler (.zcc_sanitize_trap)
Trap #5 — Label collision. Every injected check jumps to a crash handler. This must be ONE shared function emitted once into the preamble — NOT a per-check label. If you generate a new label per injection site, the label counter diverges between stage2 and stage3, breaking the Golden Rule even with the flag off.
Crash Handler Architecture
  • Emit .zcc_sanitize_trap: ONCE at top of output file when cc->sanitize == 1 — before any function code
    Single shared target. All injection points jump here.
    part4.c codegen_preamble() CRITICAL — emit once only
  • Handler body: write diagnostic string to stderr (fd=2) via write syscall, then call abort (SIGABRT) or emit ud2 (illegal instruction trap)
    part4.c codegen_preamble() LOW
  • Diagnostic string: "ZCC SANITIZER: out-of-bounds write\n" embedded as .ascii in .rodata
    part4.c preamble LOW
; Emitted ONCE at top of output .s when sanitize=1
    .section .rodata
.zcc_san_msg:
    .ascii "ZCC SANITIZER: out-of-bounds write\n"
.zcc_san_msg_end:
    .text

.zcc_sanitize_trap:
    ; write(2, .zcc_san_msg, 36) via syscall
    movq  $1,  %rax             ; syscall: write
    movq  $2,  %rdi             ; fd: stderr
    leaq  .zcc_san_msg(%rip), %rsi
    movq  $36, %rdx             ; len
    syscall
    ; abort via SIGABRT
    movq  $60, %rax             ; syscall: exit
    movq  $134, %rdi            ; exit code 134 = SIGABRT convention
    syscall
    ud2                          ; unreachable — illegal instr if syscall fails
Phase 4 — Injection Sites
Trap #4 — Read vs Write scope. ND_DEREF on the RHS of an expression is a READ. Only inject checks for WRITES — ND_ASSIGN where the LHS is a pointer dereference. Injecting on reads too doubles instrumentation density with no additional safety guarantee for the canonical overflow class.
Primary Injection Site — ND_ASSIGN pointer writes
  • In codegen_expr() ND_ASSIGN case: after evaluating LHS address into %rax, before the actual write, check if (cc->sanitize) and emit the bounds check sequence
    part4.c ND_ASSIGN PRIMARY SITE
  • Scope guard: only inject when LHS node type indicates a pointer target — skip for stack-slot direct assignments (int x = 5)
    If ZCC's type system doesn't track this cleanly, use a conservative heuristic: inject on all ND_ASSIGN where LHS contains ND_DEREF.
    part4.c MEDIUM
  • Register preservation: the check must not clobber %rax (which holds the address being checked). Save %rax to %rcx before cmp, or use %rax directly in cmpq (safe — cmpq is non-destructive)
    cmpq is read-only on both operands. %rax survives the check intact.
    part4.c LOW — cmpq non-destructive
  • Secondary site: ND_DEREF on LHS of compound assignment operators if ZCC supports them
    part4.c ND_DEREF LOW
/* part4.c — ND_ASSIGN injection skeleton */

    fprintf(cc->out, "    movq %%rax, (%%rcx)\n"); /* actual write */

; Step 1: evaluate destination address → %rcx (via normal LHS eval + pushq/popq)

PRIME Energy Analysis
H₀ = 0.28 — lowest cascade energy of all T5–T8 vectors. T8 is orthogonal to the optimization chain: the injected cmpq + ja sequence is new assembly that doesn't touch any existing codegen path. The only self-hosting risk is Trap #3 (flag leakage) and Trap #5 (label collision) — both resolved architecturally at the flag guard and preamble-only label emission. Stage2 ≡ Stage3 is mathematically guaranteed when the flag is off.
Strategy Decision
Recommendation: Strategy A (%r15 register) + ND_DEREF LHS scope guard.
Strategy A gives 1-cycle check overhead vs Strategy B's memory load. The %r15 push/pop in the prologue/epilogue is standard callee-save — identical pattern to what T7 (register harvesting) will add for %r12/%r13. Implementing Strategy A now creates the prologue/epilogue infrastructure that T7 reuses, making T8 a force multiplier for the T7 buildout. Scope guard (only inject on ND_DEREF LHS) eliminates the stack aliasing trap entirely without needing a two-sided range check.
