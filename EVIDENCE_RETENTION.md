# Evidence Retention Policy

This document defines the lifecycle, retention periods, and verification procedures for artifacts generated by the Sandbox Mutagenesis pipeline.

## Retained Artifacts
The following artifacts MUST be retained for every scheduled execution block:
- `release_artifacts/` bundle
- `sandbox_canary_report.json`
- `sandbox_attestation.json`
- `release_gate_report.json`
- `metrics.json`
- `sbom.json`
- `provenance.json`
- `incident_drill_results.json`
- `chimera_run_report.first.json`
- `chimera_run_report.second.json`
- `manifest.json`

## Retention Period
- **Development/Smoke Runs:** 7 Days
- **Pilot Schedule Runs:** 90 Days
- **Quarantined Artifacts:** Indefinite (Requires manual deletion by a Security Operator)

## Verification
Any artifact retrieved from cold storage MUST be verified using `verify_release_artifacts.py` to assert the hash integrity of the bundle against `manifest.json`.

## Deletion
- Standard aging artifacts are deleted automatically after 90 days.
- Quarantined artifacts require an explicit security sign-off before purging.
